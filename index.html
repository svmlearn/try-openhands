播放板块

主播放界面

主播放界面提供音乐播放控制功能，包括播放/暂停、上一首/下一首、音量调节等。用户可以查看当前播放歌曲的信息，如封面、歌名、歌手等。

  • 播放控制（播放/暂停/上一首/下一首）
  • 音量调节和播放进度控制
  • 歌曲信息展示（封面、歌名、歌手）
  • 播放模式切换（单曲循环/随机播放等）
  • 蓝牙设备连接功能

排行榜板块

排行榜——舞队列表与搜索

排行榜版块（原社区版块）主要帮助用户发现和加入合适的舞队。由于原SaaS架构调整为非SaaS架构，舞队数据失去社区维度，现通过地理位置和舞种进行组织。

核心功能：

  • Tab切换：舞队列表 + 活动中心（预留）
  • 地区筛选：省市区三级联动选择器 + 一键定位
  • 舞种筛选：多选舞种类型（广场舞、民族舞等）
  • 搜索功能：支持舞队名称和领队姓名模糊搜索
  • 舞队列表：头像、名称、领队、成员数、地点、舞种标签
  • 一键加入：直接申请加入舞队

技术要点：

  • 三级级联选择器：省份选择后自动更新城市，城市选择后更新区域
  • 统计信息智能更新：根据选择层级显示对应统计数据
  • 搜索优化：支持拼音首字母，实时搜索提示
  • 活动中心Tab：占位"敬请期待"，为积分体系预留

舞队详情

未加入视图

舞队详情页面为未加入用户提供舞队基本信息展示和一键申请加入功能。采用适老化设计，操作简单直观，无需填写复杂表单。

  • 舞队基本信息（头像、名称、舞种、成员数等）

  • 实时状态反馈：按钮状态自动变为"已申请"
  • 成功提示：显示"申请已提交，等待领队审批"

舞队详情页采用重构设计，删除独立管理页面，将管理功能整合到详情页中。根据用户角色（领队/普通成员）展示不同功能权限。

领队功能权限：

  • 快速操作区：舞队二维码、扫码报修、查看成员、待审批
  • 更多菜单：成员管理、设备管理、修改舞队信息、转让领队、解散舞队

普通成员功能：

  • 快速操作区：舞队二维码、扫码报修
  • 更多菜单：退出舞队

设备管理（核心业务逻辑）

设备管理功能经过重新设计，从"编辑时段"改为"查看使用时段"的只读模式，基于后台预设的开关机模版进行时间段申请。

核心逻辑变更：

  • 开关机模版系统：后台管理预设设备开放时间，舞队从模版中选择
  • 占用校验机制：其他舞队已勾选的时间段显示灰色，不可选择
  • 只读时段查看："编辑时段" → "查看使用时段"，仅展示已申请时间
  • 申请审核流程：待激活 → 已激活的管理员审核机制

数据结构分离：

  • deviceTemplates：设备开放时间模版（后台设置）
  • teamSchedules：舞队已申请的时间段记录
  • deviceApplications：设备申请审核状态记录

核心功能流程：

  • 扫码绑定：扫描设备二维码 → 选择预设时间段 → 提交申请
  • 占用展示：已占用时间显示灰色 + 占用方信息
  • 状态管理：设备卡片显示"待激活"/"已激活"状态
  • 操作权限：查看使用时段 + 解绑/取消申请

创建舞队功能

用户可以创建新舞队，成为舞队领队。创建舞队功能专注于收集完整信息以支持排行榜筛选功能。加入舞队功能已整合到舞队详情页中。

加入舞队流程：

  • 在排行榜页面浏览舞队，点击进入舞队详情页
  • 在舞队详情页点击"加入舞队"按钮
  • 直接提交申请，按钮变为"已申请"状态
  • 等待领队审批，无需填写额外表单

创建舞队（PRD核心要求）：

  • 基本信息：舞队名称（必填，全局唯一）、头像（选填）、简介（必填）、舞种选择（最多3个）
  • 活动地点（必填）：三级联动地区选择 或 一键定位获取到区级别地址
  • 数据校验：舞队名称实时校验、必填项验证
  • 用户体验：适老化设计、操作简单顺畅

技术要点：

  • 舞队名称唯一性校验、位置解析接口
  • 创建成功后自动成为领队，跳转舞队管理页
  • 支持图片上传（头像）和多选舞种（最多3个）
  • 加入舞队功能已整合到team-detail-guest.html

个人中心板块

个人中心板块提供用户个人信息管理和各类申请记录查看功能，采用标准小程序UI设计，符合中老年用户使用习惯。

核心功能：

  • 用户信息：头像、昵称、个人设置管理
  • 舞队申请记录：创建舞队的审批状态（审批中、已通过、已拒绝）
  • 入队申请记录：申请加入舞队的状态，支持取消申请操作
  • 标准UI设计：88px小程序顶部覆盖层，底部导航栏规范

技术要点：

  • 页面内容从88px下方开始显示（小程序UI规范）
  • 菜单项点击动画效果，统一样式规范
  • 完整的CSS重置样式，确保100%契合小程序框架
